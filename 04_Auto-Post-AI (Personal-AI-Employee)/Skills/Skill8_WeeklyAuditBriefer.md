# Skill 8: Weekly Audit Briefer

## Purpose

Runs every Sunday to audit the previous week's activity across the entire vault.
Reads `/Done/`, `/Logs/`, `Company_Handbook.md`, and `Business_Goals.md` (if present),
applies pattern matching for revenue, subscriptions, and expenses, then produces a
CEO-ready briefing in `/Briefings/ceo_briefing_[YYYY-MM-DD].md`.

This is a **Gold Tier autonomous skill** — it executes on a schedule without human
intervention. The output briefing is human-readable and requires no further processing.

---

## Output Location

```
/Briefings/ceo_briefing_YYYY-MM-DD.md
```

Duplicate-safe: if a briefing already exists for the date, appends `_N` suffix.

---

## Briefing Template

```markdown
---
type: ceo_briefing
period: "YYYY-MM-DD to YYYY-MM-DD"

---

# CEO Briefing — Week of YYYY-MM-DD

## Executive Summary
(3-4 sentence high-level overview of the week)

## Revenue & Financials
(Subscriptions, payments, expenses detected via pattern matching)

## Completed Tasks
(Table: Task | Type | Channel | Date Completed)

## Bottlenecks
(Items stuck in pipeline, rejected tasks, unresolved Needs Action items)

## Suggestions
(3-5 strategic recommendations based on the audit)

---
*Auto-generated by Skill 8: Weekly Audit Briefer — Gold Tier AI Employee*
```

## Handbook Rules Applied

- **Payments > $500** — flagged individually in Revenue section with `⚠` warning.
- **Politeness check** — briefing tone is professional, objective, not alarmist.
- **Business Goals** — if `Business_Goals.md` exists, suggestions are aligned to goals.

---

## Pattern Matching

### Revenue Signals
| Pattern | Example |
|---------|---------|
| Dollar amounts in Done/Logs files | `$1,200`, `$350.00` |
| Keywords near amounts | `invoice`, `payment received`, `sale`, `earned`, `revenue` |
| Log entries with `status: sent` or `status: executed` | LinkedIn posts, email campaigns |

### Subscription / Expense Signals
| Pattern | Example |
|---------|---------|
| Keywords | `subscription`, `renewal`, `monthly fee`, `annual plan`, `cost`, `expense` |
| Dollar amounts in Logs files paired with expense keywords | `$99/month`, `$1,200/year` |
| Rejected or returned payments | `refund`, `chargeback`, `cancelled` |

### Bottleneck Signals
| Signal | Detected From |
|--------|---------------|
| Files still in `/Needs Action/` older than 2 days | `mtime` check |
| Files in `/Rejected/` this week | Rejected folder scan |
| Tasks with `priority: high` that are not in `/Done/` | Cross-reference |
| Repeated same-type failures in Logs | Log pattern scan |

---

## Procedure

### Step 1: Load Context Files
1. Read `Company Handbook.md` → cache rules (payment threshold, politeness).
2. Read `Business_Goals.md` → cache goals for suggestion alignment (warn if missing, use defaults).
3. Record week window: `[last Sunday to today]`.

### Step 2: Scan `/Done/`
- List all `.md` files in `/Done/`.
- Parse YAML frontmatter: `type`, `status`, `created`, `priority`, `channel`.
- Extract any dollar amounts found in file body.
- Categorise as: task, email, linkedin_post, payment, generic.

### Step 3: Scan `/Logs/`
- List all `.md` files in `/Logs/`.
- Apply revenue regex: `\$\s?[\d,]+\.?\d*` near revenue keywords.
- Apply expense regex: same pattern near expense keywords.
- Count log events by type.

### Step 4: Scan Bottleneck Sources
- `/Needs Action/` — files older than 48 hours.
- `/Rejected/` — any `.md` files created this week.
- `/Pending Approval/` — files older than 24 hours.

### Step 5: Synthesise Briefing

Use the collected data to fill the CEO briefing template:

1. **Executive Summary** — counts + highlights in plain English.
2. **Revenue & Financials** — table of amounts, flag > $500 per Handbook.
3. **Completed Tasks** — table from `/Done/` scan.
4. **Bottlenecks** — table from bottleneck scan.
5. **Suggestions** — rule-based recommendations (see table below).

### Suggestion Rules

| Condition | Suggestion |
|-----------|-----------|
| Rejected tasks > 2 | "Review rejection reasons and adjust content strategy." |
| Pending Approval > 5 | "Clear the approval backlog — N items awaiting review." |
| Revenue < last week's detected revenue | "Revenue declined — investigate pipeline health." |
| No Done items this week | "No tasks completed — check if watchers are running." |
| Expense keywords detected | "Review subscription costs against Business Goals." |
| LinkedIn posts > 3 | "Strong social activity — consider tracking engagement metrics." |
| High-priority Needs Action > 3 | "N high-priority items unaddressed — escalate immediately." |

### Step 6: Write Briefing File

Write to `/Briefings/ceo_briefing_[YYYY-MM-DD].md`.
Create `/Briefings/` directory if it does not exist.
Output the file path on success.

### Step 7: Output Summary

```
WEEKLY AUDIT BRIEFER COMPLETE
==============================
Week Period   : YYYY-MM-DD to YYYY-MM-DD
Done Items    : N
Revenue Found : $X,XXX.XX
Expenses Found: $X,XXX.XX
Bottlenecks   : N
Briefing Path : Briefings/ceo_briefing_YYYY-MM-DD.md
==============================
```

## Scheduler Integration

This skill is integrated into both schedulers with a **day-of-week guard** (Sunday only):

| Scheduler | Guard Condition | Cron equivalent |
|-----------|----------------|-----------------|
| `daily_scheduler.sh` | `[ "$(date +%u)" = "7" ]` | `0 8 * * 0` |
| `daily_scheduler.ps1` | `(Get-Date).DayOfWeek -eq 'Sunday'` | Windows Task Scheduler weekly trigger |

The daily scheduler calls the weekly audit briefer **after** the daily briefing on Sundays.

---

## Error Handling

| Condition | Action |
|-----------|--------|
| `/Done/` missing | Log warning, continue with empty task list |
| `/Logs/` missing | Log warning, continue with empty log list |
| `Business_Goals.md` missing | Warn, use generic suggestions |
| `Company Handbook.md` missing | Warn, apply default rules |
| `/Briefings/` missing | Create directory automatically |
| Duplicate briefing for date | Append `_N` counter to filename |
| No data found at all | Write briefing with "No activity recorded this week" |

## Python Implementation

```bash
# Run manually (generates briefing for today's week)
python Skills/weekly_audit_briefer.py

# Dry run — analyse and print, no files written
python Skills/weekly_audit_briefer.py --dry-run

# Specify a custom reference date (for backfills)
python Skills/weekly_audit_briefer.py --date 2026-02-16

# With PM2 (runs once, exits — scheduler calls it on Sunday)
pm2 start Skills/weekly_audit_briefer.py --interpreter python --name weekly-audit
```
