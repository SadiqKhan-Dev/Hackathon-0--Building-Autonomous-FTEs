# Skill 7: Twitter Post Generator

## Purpose
Process Twitter/X items from `/Needs Action/` (`TWITTER_*.md`), generate an enhanced
summary, score each item as a business lead, draft a public tweet (280 characters max)
and a private DM reply for high/medium leads, save to `/Plans/twitter_draft_[date].md`,
and route to `/Pending Approval/` for Human-in-the-Loop (HITL) review.

This skill is the downstream processor for the **Twitter / X Watcher**.

---

## Source Files

| File Prefix | Platform | Content Types |
|-------------|----------|---------------|
| `TWITTER_*` | Twitter / X | dm, notification, tweet |

Only `TWITTER_*.md` files are processed. Other `/Needs Action/` files are skipped.

---

## Tweet Character Budget

| Draft Type | Limit | Strategy |
|------------|-------|----------|
| Public tweet | 280 characters | Concise template + max 2 hashtags |
| DM reply | 10,000 characters | Full warm message, no limit concerns |
| Quote-tweet | 280 characters | Context prefix + link shortened |

The skill validates every tweet draft against the 280-character limit and
truncates with `...` if needed before saving.

---

## Handbook Rules (from `/Company Handbook.md`)
- **Always be polite** -- warm, benefit-focused tone; no aggressive sales language.
- **Payments > $500** -- escalate to HITL with `priority: high` regardless of lead score.
- **No personal data** -- never include the sender's private info in a public tweet draft.
- **Soft call-to-action only** -- "DM me", "Let's connect", never "Buy now" or pressure phrases.

---

## Procedure

### Step 1: Load Company Handbook
Read `/Company Handbook.md` and cache payment threshold and tone rules.

### Step 2: Scan `/Needs Action/` for Twitter Files
- List all `.md` files starting with `TWITTER_`
- For each, read YAML frontmatter:
  - `content_type` (dm / notification / tweet)
  - `from`, `handle` (@username)
  - `keyword_matched`
  - `summary` (pre-generated by watcher)
  - `priority`
- Read the full content preview from the file body

If no `TWITTER_*` files exist, output:
```
NO TWITTER ITEMS: No Twitter files in /Needs Action. Ensure twitter_watcher.py is running.
```
and stop.

### Step 3: Generate Enhanced Summary
For each item, produce a structured 4-part summary:

```
SUMMARY
-------
Platform   : Twitter / X
Type       : dm | notification | tweet
From       : [sender display name]
Handle     : @[username]
Keyword    : [matched keyword]
Excerpt    : [first 2 sentences of content]
Sentiment  : positive | neutral | urgent
Lead Score : HIGH | MEDIUM | LOW
Char Count : [tweet draft character count] / 280
```

**Sentiment rules:**
- `urgent`, `asap`, `immediately`, `need`, `deadline` in content -> urgent
- `interested`, `looking for`, `love to`, `keen`, `would like` -> positive
- Default -> neutral

**Lead score rules:**
- keyword `sales` or `client`, any sentiment -> HIGH
- keyword `project` + sentiment positive or urgent -> MEDIUM
- keyword `project` + sentiment neutral -> LOW
- Default -> LOW

### Step 4: Route by Lead Score

#### 4a -- HIGH or MEDIUM: Draft Tweet + DM Reply
1. **Public tweet draft** (max 280 chars):
   ```
   Excited to offer [service] for [benefit]!
   Looking for [outcome]? Let's connect.
   DM me to get started! #[tag1] #Business
   ```
   - Validate character count -- trim service description if needed
   - Never include sender's @handle in the public tweet

2. **DM reply draft** (private, warmer tone):
   ```
   Hi [first name]! Thanks for your [dm/tweet/notification] about [subject].
   I would love to help with [service].
   Can we set up a quick call to discuss your needs?
   Feel free to DM me anytime -- happy to connect!
   ```

3. Save both to `/Plans/twitter_draft_[date]_[sender-slug].md`
4. Copy to `/Pending Approval/` for HITL
5. Log: `TWITTER DRAFT CREATED -- [filename]`

#### 4b -- LOW: Summary Only
1. Write enhanced summary only
2. Save to `/Plans/twitter_draft_[date]_[sender-slug].md`
3. Mark `status: needs_review` (no response drafted)
4. Still copy to `/Pending Approval/` for human decision

### Step 5: Save Draft to `/Plans/`
Create: `/Plans/twitter_draft_[YYYY-MM-DD]_[sender-slug].md`

```yaml
---
type: twitter_draft
platform: twitter
content_type: [dm | notification | tweet]
source_file: "Needs Action/[source filename]"
from: "[display name]"
handle: "@[username]"
keyword_matched: "[keyword]"
lead_score: [high | medium | low]
sentiment: [positive | neutral | urgent]
has_tweet_draft: [true | false]
has_dm_draft: [true | false]
tweet_char_count: [N]
priority: [high | medium | low]
status: pending_approval
created: "[YYYY-MM-DD HH:MM:SS]"
source_skill: Skill7_TwitterPostGenerator
payment_flag: [true | false]
---
```

### Step 6: HITL Gate -- `/Pending Approval/`
- Copy from `/Plans/twitter_draft_[date].md` to `/Pending Approval/`
- Status set to `pending_approval`
- Do NOT tweet or send DM -- wait for human approval
- Skill 4 (HITL Handler) executes on approval

### Step 7: Output Result
```
TWITTER POST GENERATOR COMPLETE
================================
Platform    : Twitter / X
Source      : /Needs Action/TWITTER_[timestamp]_[sender].md
Type        : [dm | notification | tweet]
Lead Score  : [HIGH | MEDIUM | LOW]
Tweet chars : [N]/280
Has Tweet   : Yes / No
Has DM      : Yes / No
Draft       : /Plans/twitter_draft_[date]_[sender].md
Pending     : /Pending Approval/twitter_draft_[date]_[sender].md
Status      : Awaiting human approval (HITL)
Next Step   : Review /Pending Approval/ -> move to /Approved to publish
================================
```

## Tweet Draft Validation

| Check | Rule |
|-------|------|
| Character count | Must be <= 280 before saving |
| Hashtag count | Maximum 2 hashtags per tweet |
| @mentions | NOT included in public tweet (privacy) |
| URLs | Counted as 23 chars each (Twitter t.co wrapping) |
| Emoji | Each emoji counts as 2 chars; avoid unless natural |
| Payment reference | Removed from public tweet if amount > $500 |

---

## Lead Score / Routing Matrix

| Content Type | Keyword | Sentiment | Lead Score | Tweet | DM Reply |
|--------------|---------|-----------|------------|-------|----------|
| DM | sales | any | HIGH | Yes | Yes |
| DM | client | any | HIGH | Yes | Yes |
| DM | project | positive/urgent | MEDIUM | Yes | Yes |
| DM | project | neutral | LOW | No | No |
| Notification | sales | positive | HIGH | Yes | No |
| Notification | client | any | HIGH | Yes | No |
| Tweet | sales | any | HIGH | Yes | No |
| Tweet | project | positive | MEDIUM | Yes | No |
| any | any | neutral (no kw) | LOW | No | No |

> DMs always get a DM reply draft. Notifications and tweets get a public tweet draft only.

---

## Error Handling

| Condition | Action |
|-----------|--------|
| No TWITTER_* files in /Needs Action | Log message and exit cleanly |
| Tweet draft exceeds 280 chars | Trim service description, retry, hard-truncate |
| Payment > $500 in content | Force `priority: high`, flag in YAML |
| /Plans or /Pending Approval missing | Create folder, continue |
| Duplicate draft same sender same day | Append `_N` counter to filename |
| Handbook unavailable | Warn and apply default polite rules |
| Missing YAML frontmatter in source | Parse from filename, continue |

## Python Implementation

```bash
# Process all TWITTER_* files in /Needs Action (default)
python Skills/twitter_post_generator.py

# Dry run -- summarise and score only, no files written
python Skills/twitter_post_generator.py --dry-run

# Process a single specific file
python Skills/twitter_post_generator.py --file "Needs Action/TWITTER_20260222_120000_JohnDoe.md"

# With PM2
pm2 start Skills/twitter_post_generator.py --interpreter python --name twitter-generator
pm2 save
```
