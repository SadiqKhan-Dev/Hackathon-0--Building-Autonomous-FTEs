# Skill 6: Social Summary Generator

## Purpose
Process Facebook and Instagram items from `/Needs Action/`, generate a structured
summary of each message or post, draft a polished response or outbound post when a
sales lead is detected, save the draft to `/Plans/`, and route it to `/Pending Approval/`
for Human-in-the-Loop (HITL) review before any response is sent.

This skill is the **downstream processor** for the Facebook & Instagram Watcher. It
converts raw social media items into actionable business drafts.

---

## Source Files

| File Prefix | Platform | Content Types Handled |
|-------------|----------|-----------------------|
| `FACEBOOK_*` | Facebook | Messenger DM, notification |
| `INSTAGRAM_*` | Instagram | Direct message, activity notification |

Only files with these prefixes are processed. Other files in `/Needs Action/` are ignored.

---

## Handbook Rules (from `/Company Handbook.md`)
- **Always be polite** — all drafted responses are warm, benefit-focused, never pushy.
- **Payments > $500** — escalate to HITL with `priority: high` before drafting any post.
- **No aggressive sales language** — soft calls-to-action only (DM, connect, reply).
- **Confidential data** — never include personal information from the source message in a public post draft.

---

## Procedure

### Step 1: Load Company Handbook
Read `/Company Handbook.md` and extract active rules for tone and payment thresholds.

### Step 2: Scan `/Needs Action/` for Social Files
- List all `.md` files in `/Needs Action/` starting with `FACEBOOK_` or `INSTAGRAM_`
- For each file, read YAML frontmatter and extract:
  - `platform` (facebook / instagram)
  - `content_type` (message / notification / post)
  - `from` (sender)
  - `keyword_matched`
  - `summary` (pre-generated by watcher)
  - `priority`
- Read the full content preview from the file body

If no matching files are found, output:
```
NO SOCIAL ITEMS: No Facebook or Instagram files in /Needs Action.
```
and stop.

### Step 3: Generate Enhanced Summary
For each file, produce a structured 3-part summary:

```
SUMMARY
-------
Platform : Facebook | Instagram
Type     : message | notification | post
From     : [sender name]
Keyword  : [matched keyword]
Excerpt  : [first 2 sentences of the content]
Sentiment: [positive | neutral | urgent] (inferred from language)
Lead Score: [high | medium | low] (based on keyword + urgency)
```

**Sentiment inference rules:**
- `urgent`, `asap`, `immediately`, `need` in content -> urgent
- `interested`, `love`, `great`, `looking for` -> positive
- Default -> neutral

**Lead score rules:**
- keyword = `sales` + sentiment = positive or urgent -> high
- keyword = `client` -> high
- keyword = `project` -> medium
- Default -> low

### Step 4: Route by Lead Score

#### 4a — High / Medium Lead Score (Draft Response + Post)
1. Draft a Facebook/Instagram reply message:
   ```
   Hi [sender]! Thank you for reaching out about [subject].
   I would love to help you with [service]. Let us set up a quick call to discuss.
   Feel free to DM me anytime — always happy to connect!
   ```
2. Draft an outbound social post (for Instagram feed or Facebook page):
   ```
   [Same template as Skill 3 LinkedIn post, adapted for social tone]
   Excited to help with [service/keyword topic]!
   DM me to get started. #[keyword] #Business
   ```
3. Save both to `/Plans/facebook_draft_[YYYY-MM-DD].md`
4. Route to `/Pending Approval/` — HITL required before posting

#### 4b — Low Lead Score (Summary Only)
1. Generate the enhanced summary only
2. Save a brief summary note to `/Plans/facebook_draft_[YYYY-MM-DD].md`
3. Mark as `status: needs_review` — no response drafted, human decides

### Step 5: Save Draft to `/Plans/`
Create file: `/Plans/facebook_draft_[YYYY-MM-DD].md`

```yaml
---
type: facebook_draft
platform: [facebook | instagram]
content_type: [message | notification | post]
source_file: "Needs Action/[source filename]"
from: "[sender]"
keyword_matched: "[keyword]"
lead_score: [high | medium | low]
has_response_draft: [true | false]
priority: [high | medium | low]
status: pending_approval
created: "[YYYY-MM-DD HH:MM:SS]"
source_skill: Skill6_SocialSummaryGenerator
---

# Social Draft: [platform] - [YYYY-MM-DD]

## Summary

[enhanced 3-part summary]

## Response Draft (Reply to Sender)

[drafted reply message — if lead_score is high or medium]

## Post Draft (Outbound Feed Post)

[drafted social media post — if lead_score is high or medium]

## Checklist

- [ ] Review tone against Company Handbook (always polite)
- [ ] Verify response is appropriate for the sender
- [ ] Confirm no confidential data in the post draft
- [ ] Move to /Approved to send / post
- [ ] Move to /Rejected to discard
```

### Step 6: HITL Gate — Route to `/Pending Approval/`
- Copy the draft from `/Plans/facebook_draft_[date].md`
  to `/Pending Approval/facebook_draft_[date].md`
- Update `status: pending_approval`
- Do NOT send or post — wait for human approval
- HITL Handler (Skill 4) executes on approval

### Step 7: Output Result
```
SOCIAL SUMMARY GENERATOR COMPLETE
Platform     : Facebook | Instagram
Source       : /Needs Action/[filename]
Lead Score   : high | medium | low
Draft Saved  : /Plans/facebook_draft_[date].md
Pending Path : /Pending Approval/facebook_draft_[date].md
Status       : Awaiting human approval (HITL)
Next Step    : Review /Pending Approval/ and move to /Approved to publish
```

## Lead Score Reference

| Keyword | Sentiment | Lead Score | Action |
|---------|-----------|------------|--------|
| sales   | positive / urgent | high | Draft reply + post + HITL |
| client  | any | high | Draft reply + post + HITL |
| project | positive / urgent | medium | Draft reply + post + HITL |
| project | neutral | low | Summary only -> needs_review |
| any     | neutral (no urgency) | low | Summary only -> needs_review |

---

## Error Handling

| Condition | Action |
|-----------|--------|
| `/Needs Action` has no FB/IG files | Log `NO SOCIAL ITEMS` and exit cleanly |
| File missing YAML frontmatter | Use filename to extract fields, continue |
| Payment > $500 detected in content | Force `priority: high`, escalate to HITL |
| `/Plans` or `/Pending Approval` missing | Create folder, continue |
| Duplicate draft for same day | Append `_N` counter to filename |
| Handbook unavailable | Warn and apply default polite rules |
